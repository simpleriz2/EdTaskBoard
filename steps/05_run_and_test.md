# Шаг 5: Запуск и проверка (Инструкция для ИИ-агента)

**КОНТЕКСТ ДЛЯ АГЕНТА:** Единая точка входа — `http://localhost` (Caddy). Не обращайся напрямую к портам 3000 или 3001 в тестах.

## 5.1 Запуск
```bash
docker-compose up -d --build
```
Проверь, что все 4 сервиса запустились:
```bash
docker-compose ps
```
Ожидаемый результат: все сервисы в статусе `Up` — `app`, `server`, `caddy`, `turnserver`.

Валидация сети Caddy:
```bash
# Фронтенд доступен
curl -I http://localhost
# Сервер доступен через Caddy
curl http://localhost/health
```
Оба запроса должны вернуть `200 OK`.

## 5.2 Проверка через Browser Subagent (авто-валидация)

Для каждого шага используй инструмент `browser_subagent`.

**Шаг A — Lobby:**
- Открой `http://localhost/room/test-room-1`
- Убедись, что рендерится элемент `<input>` для имени и `<video>` для превью камеры
- Введи имя "Agent One", кликни "Join Room"
- Убедись, что после клика отрендерился `<canvas>` (tldraw)

**Шаг B — Синхронизация доски (2 вкладки):**
- Открой вторую вкладку → `http://localhost/room/test-room-1` → введи "Agent Two", войди.
- В первой вкладке нарисуй прямоугольник (симулируй mouse events на canvas).
- Убедись, что во второй вкладке появился тот же элемент.
- Проверь Network в DevTools: должны быть активные WebSocket фреймы к `ws://localhost/tldraw-sync/...`.

**Шаг C — Persistence:**
```bash
docker-compose restart server
```
- Перезагрузи страницу в браузере.
- Убедись, что нарисованные фигуры сохранились.

**Шаг D — Video (WebRTC):**
- В обеих вкладках должны быть видны плавающие карточки с видео.
- Попробуй перетащить карточку в другой угол — она должна перемещаться.
- Кликни "Cam" → видео должно пропасть из карточки на экране другого участника.
- Проверь логи coturn:
```bash
docker-compose logs turnserver | grep "session"
```
